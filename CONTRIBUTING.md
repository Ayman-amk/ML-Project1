# Contributing

## Commits

Use clear prefixes:

-   `feat:` new feature
-   `fix:` bug fix
-   `chore:` tooling, scaffolding
-   `docs:` README/docs only
-   `refactor:` code moves / no behavior change

## Branches

-   `main` is protected
-   Use short feature branches: `feat/data-loader`, `feat/tf-training`

## PRs

-   Keep scope focused, describe testing steps, link to MLflow run if useful.

## Style

-   Python 3.10+, run modules from repo root via `python -m src....`
